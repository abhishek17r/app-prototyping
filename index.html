Proto-Kit

A self-contained kit for building mobile-app feature prototypes that match a real recording at 1:1 fidelity. Phone shell, bottom sheet, list state — themeable for any product via CSS variables and one ProtoKit.init({}) call.

Live prototypes

Starter demo
Blank phone shell wired up with the "Add to list" pattern. Copy and re-skin.

Build your own

cp apps/starter.html apps/spotify.html
# Then edit ProtoKit.init({ ... }) at the bottom:
#   brand:   { name, color }
#   item:    { id, title }   ← the thing being added
#   defaultList:  { id, name, color }
#   customLists:  [{ id, name, color }, ...]
#   unit:    "songs" | "movies" | "places" | ...

The body of the phone is yours — header, hero, content, action row. The lib injects the sheet, persists state per brand in localStorage, and wires up the trigger button by id (#addToListBtn).

Extract frames from a screen recording

pip3 install --user imageio imageio-ffmpeg pillow
python3 extract-frames.py path/to/recording.mp4 --fps 1 --out frames/

Outputs t000.jpg, t001.jpg, … one frame per second by default. Use --fps 2 for denser sampling at screen transitions.

Serve locally

./go.sh                    # python3 -m http.server 4477
# then open http://localhost:4477

Layout

  • index.html — this page
  • go.sh — start the local server
  • extract-frames.py — video → frames
  • lib/shell.css — phone frame, status bar, bottom nav, action row
  • lib/sheet.css — bottom sheet, list rows, create form, toast
  • lib/state.js — list state + localStorage persistence
  • lib/sheet.js — sheet open/render/toggle/create
  • apps/*.html — one file per prototype
